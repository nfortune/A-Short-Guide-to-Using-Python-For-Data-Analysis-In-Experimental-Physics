If using the abbreviated prefix  np instead of numpy doesn't make you swoon, you don't have to use it, but its usage is ubiquitous. To skip it, type import numpy instead of import numpy as np. In that case, you would then precede the loadtxt command with numpy., as in numpy.loadtxt. If you preferred whimsy over clarity, you could try import numpy as turtles.  Don't! 
We call this the traditional method because this is what you will find in the examples included in the official numpy user guide and quickstart tutorials.  If you use this method, take note that you will  have to specify that you are using a function  or module from the numpy package each time you use that function by preceding it either with numpy or the abbreviation of your choice (such as np).  This is why the loadtxt command in this method is written np.loadtxt instead of loadtxt.  
In both cases, we used the loadtxt command from Numerical Python (numpy) to load the data from a .csv file. loadtxt can also load other types of text files: see the numpy.loadtxt manual page for details. 
Note for Python geeks: if you are reading this, you probably know that if you use the direct method to type from numpy import * then  you can import all numpy commands at once instead of one at a time  .This import * approach is convenient — we will sometimes do it ourselves, being not as pure in thought as others might be on this issue— but take two considerations in mind: (1) you are importing code you don't need (2) you will overwrite an functions with the same name as a numpy function with the numpy version.
