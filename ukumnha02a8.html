We call this the traditional method because this is what you will find in the examples included in the official numpy user guide and quickstart tutorials.  If you use this method, take note that you will  have to specify that you are using a function  or module from the numpy package each time you use that function. To do so, precede the name of the function with np.  This is why the loadtxt command in this method is written np.loadtxt instead of loadtxt.  In examples, the use of the  np.prefix helps identify when the numpy package is being used but in your own code it can be a nuisance to remember! 
The choice of methods is yours, as long as you are consistent in your usage.
Note for Python geeks: if you are reading this, you probably kto import all modules from numpy at once using the direct method,  type from numpy import * (in place offrom numpy import loadtxt). Although a convenient and common practice, this import * approach has at least two drawbacks: (1) you are importing code you don't need (2) that additional code might include functions with the same name as one you are already using (which would overwrite your earlier definition). Because it also imports all modules at once, the traditional method  import numpy as np command has these same drawbacks.
