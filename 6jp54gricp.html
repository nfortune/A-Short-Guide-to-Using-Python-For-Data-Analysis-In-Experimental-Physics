handling uncertainties in Uncertainties

For propagation of uncertainty calculations without units , we use  Uncertainties: a Python package written by Eric. O. Lebigot.  The uncertainties module returns its result with the uncertainty specified by linear error propagation theory, correctly taking into account any direct correlations between variables.  This is in fact what Pint uses under the hood! 
If you need still more advanced approaches to propagation of uncertainty, Lebigot recommends trying  soerp (second-order approximations) and mcerp (Monte-Carlo approach).  I haven't yet done so.

Importing and exporting data from spreadsheet files

There are many ways to import, export, and represent data in files. Here we provide just enough to get you started.  To that purpose, let's assume that the data values were first entered in a spreadsheet consisting of one or more columns — one for each measured variable — and a few initial 'header' rows of text providing information about the data that we need to skip over when loading numerical values. The data was then exported as a text file. For specificity, we're going to assume the data was exported as a '.CSV' text file (where CSV stands for 'comma-separated-variables') but the Numerical Python (numpy) command we are going to use can also handle other types of simple text files as well (such as tab or space delimited).  See the numpy.loadtxt manual page for details.
We will use the loadtxt command from Numerical Python (numpy) to load the data. Below are two ways of accomplishing the same task: loading data from a .csv text file with three columns of data (say x, y, and \(\delta y\) ) and one row of text preceding the data.  Each column of data becomes a Numerical Python array. In this example, the array names are angle, V_pd, and V_pd_delta.
This first method (used, for example,  in the text Computational Physics by Mark Newman) imports  individual functions  and modules from numpy as needed.
